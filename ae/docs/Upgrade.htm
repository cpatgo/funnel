AwebDesk Email Marketing Software 7.1.0 Upgrade Guide

Thank you for choosing Email Marketing Software from AwebDesk.

This document will assist you with the upgrade process. If you have questions or need assistance at any time please visit our support center.

Follow these steps to upgrade your AwebDesk Email Marketing Software.This guide is only for you if you have already an installed copy of AEM on your server. If you are installing it for the first time, please refer to the html file install.html found in the root of this downloaded package.

  1. Unzip the downloaded ZIP file

  2. (Please skip this step(#2) if you are using v6.3.0 as its already done by you when you upgraded to v6.3.0. If you are using AEM older than 6.3.0 pl follow this step(#2). To know which version you are using , click on admin->Settings->About).
    Open serialkey.php under /upload/cache/ directory.
    Enter the license key which you received on purchasing in that file.

    So if your license key is AEMO-1234567890123(To get your license key login here and click on purchased product details.)

    define('SERIAL_KEY', 'Enter your serial Key here');

    should be edited as(no space should be there between ''):

    define('SERIAL_KEY', 'AEMO-1234567890123');
    Also set the URL of installation there. So if you are installing AEM in http://example.com/AEM

    define('AEM_BASE_URL', 'Enter your AEM URL without a trainling /');

    shoule be edited as(no ending trailing / please note):

    define('AEM_BASE_URL', 'http://example.com/AEM');

  3. V7.1.0 has new theming system. So , please delete all the files inside:

    /templates/
    /manage/templates/

  4. Upload the contents of package /upload/ including /cache/ to your web server AEM install directory replacing the old files. If you have made changes to templates files, make sure you compare the directory /templates/ inside /upload/templates/ directory,
    and /manage/templates/ inside /upload/manage/templates/classic/ directory in order to retain the changes before uploading it to server.Basically all admin initial default templates files have now been moved to /manage/templates/classic/


  5. Run the web based updater by opening the following file (which is located on your web server) in your web browser
    • http://mydomain.com/aem/manage/updater.php
    • If it says to delete lock.txt file, then delete the file lock.txt from /cache/ on your server. You maye be told to delete lock.txt file two times during the process. That is perfectly normal.
    • Refresh the page twice on first upgrade screen. Follow the onscreen instructions.That completes your upgrade! You may now login to your software application.

  6. If something goes wrong or in case you get any license error, please verify that /cache/ directory and all contents inside it is writable(777 permissions).Also try deleting key.php file from /AEM/cache/ on serve and uploading a fresh key.php from downloaded package /cache/ on server and refresh the page.
